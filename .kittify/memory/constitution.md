## Technical Standards

### Languages and Frameworks
- Python with a minimal dependency set
- FastAPI for the HTTP API layer
- scikit-learn and numpy for ML operations
- uvicorn as the ASGI server
- joblib for model serialization
- No heavyweight or unnecessary dependencies

### Testing Requirements
- **Framework**: pytest for all tests
- **Linting**: Ruff for code formatting and linting
- **Type checking**: Pyre for static type analysis
- **Security analysis**: Pysa for taint analysis and security scanning
- **Coverage target**: 80% line coverage minimum
- All new features and bug fixes must include tests

### Performance and Scale
- Lightweight local execution is the primary target
- Fast response times — the ML sidecar must not slow down the developer's workflow
- Minimal CPU and memory footprint when running alongside sigild
- Design for local-first with a path to cloud SaaS scalability

### Deployment and Constraints
- Cross-platform: Linux, macOS, Windows
- Runs on a developer's laptop alongside the sigil daemon (sigild)
- No data leaves the machine (local-only, security-first)
- Must not require GPU or specialized hardware

## Tribal Knowledge

### Team Conventions
- Small, single-feature branches
- Commit early and often
- Small, focused PRs — each PR fixes one bug or adds one feature
- Follow PEP standards (PEP 8, PEP 257, PEP 484)
- Write high-quality, readable code

### Historical Decisions
- **Local-first with cloud path**: sigil-ml is designed to run lightweight on a
  laptop but the architecture should allow for future cloud SaaS deployment and
  running at scale
- **Security-first**: No data leaves the machine. All ML processing is local-only.
  This is a core, non-negotiable principle
- **Simplicity over complexity**: Keep everything as simple as possible. Avoid
  unnecessary abstractions, over-engineering, and premature optimization
